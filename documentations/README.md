# MedTrack Documentation Index

This folder explains what was built in each project phase and the exact steps needed to run and deploy the system.

## Files in this folder

- `phases-overview.md`: what each phase delivered from Phase 1 to Phase 14
- `getting-the-system-running.md`: local setup, Firebase setup, environment variables, emulator notes, and deployment flow
- `deployment-checklist.md`: deployment order and final production-readiness checks

## Recommended reading order

1. Read `phases-overview.md` to understand the system structure.
2. Follow `getting-the-system-running.md` to configure Firebase and run the app locally.
3. Use `deployment-checklist.md` before presenting or deploying to production.
4. Use the existing QA guide in [`../docs/testing-and-qa.md`](../docs/testing-and-qa.md) for repeatable validation.
